Project Context & Overview

MiniFlow is a low-latency speech-to-speech (STS) conversational AI pipeline designed to run entirely on consumer-grade hardware with limited resources. The system processes audio input through three sequential stages to enable real-time voice conversations.

Architecture Pipeline

Stage 1: ASR

Audio → Text
Whisper-small (39M params)

Stage 2: LLM

Text → Response
Qwen2.5-3B-Instruct (4-bit NF4)

Stage 3: TTS

Text → Audio
VibeVoice-Realtime-0.5B

Success Criteria

End-to-end Latency <1s
ASR WER <10%
GPU VRAM <6GB
TTS Naturalness UTMOS >4.0

Current Baseline Performance

ASR (Whisper-small) 425ms
LLM (Qwen-3B) 36s
TTS (VibeVoice) 16s
Total ~53s

Priority: Version documentation and cross-compilation testing

Pipeline Integration Phenomena

Cold-Start Latency

Model loading overhead estimates (100-500ms) require empirical measurement

Investigation: Instrumented measurement per model size and optimization

Memory Fragmentation

Sequential loading patterns may create fragmentation affecting large model allocation

Investigation: GPU memory profiler analysis and defragmentation strategies

Micro-Batching Feasibility

Short utterance grouping potential for throughput improvement

Investigation: Micro-batching simulation with latency impact measurement

Async Processing Opportunities

Stage overlap and parallel execution potential beyond current sequential design

Investigation: Dependency graph optimization and pipeline parallelism

Conclusion

MiniFlow's <1s latency target is achievable through strategic model selection and optimization, with the most promising paths involving emerging TTS solutions like MiraTTS and Chatterbox-Turbo alongside optimized LLM inference. The current 53-second baseline can theoretically reach 600-900ms through systematic ablation studies and empirical validation on consumer hardware.
